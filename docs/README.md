# Docs

这个目录分成两类内容：

1. 面向公开阅读的项目与架构入口
2. 面向维护者的工程与发布边界说明

## First-Read Path

如果你第一次阅读这个仓库，按下面顺序进入：

1. [README.md](../README.md)
2. [architecture/README.md](./architecture/README.md)
3. [architecture/specs/README.md](./architecture/specs/README.md)
4. [architecture/plans/README.md](./architecture/plans/README.md)

## Maintainer Path

如果你在维护结构、工程规则或发布边界，再进入：

1. [development/README.md](./development/README.md)

## 目录约定

- `architecture/` 负责稳定架构说明与 phase 账本入口
- `development/` 负责工程规则、目录治理、发布边界与维护者说明
