# Docs Bridge

## Purpose

This file keeps the boundary clear:

- `docs/` is the formal human-facing documentation system.
- `AGENTS.md` is the repository-level Agent entrypoint.
- `.agent/` is the Agent workflow support system.

## Reading Order

1. `AGENTS.md`
2. `.agent/references/`
3. `docs/architecture/README.md`
4. `docs/architecture/current-architecture.md`
5. `docs/architecture/target-architecture.md`
6. `docs/history/programs/zuno-target-architecture-migration-v1/README.md`

## Rule

Write formal conclusions back to `docs/`. Keep Agent-only prompts, scripts, and navigation in `.agent/`.
