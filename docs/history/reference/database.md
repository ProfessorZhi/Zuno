# 数据库结构

本文档概述 Zuno 当前主要的数据表及其职责，便于快速理解系统如何存储用户、会话、能力配置与知识资产。

## 用户与权限

### `user`

存储基础用户信息，例如：

- 用户名
- 邮箱
- 头像
- 描述
- 密码摘要

### `role`

定义角色类型，例如管理员和普通用户。

### `user_role`

维护用户与角色之间的关联关系。

## 智能体与能力配置

### `agent`

存储智能体配置，例如：

- 名称与描述
- 绑定模型
- 系统提示词
- 是否启用记忆
- 绑定的 MCP、工具、知识库

### `mcp_agent`

这是一个**历史兼容表**，用于保存早期版本中的 MCP 服务集合配置。  
它的名字里虽然带 `agent`，但在当前主运行时里并不代表“子 Agent”。

更准确地理解，它保存的是：

- 一组 MCP 服务关联关系
- 对应名称、描述和图标

## 对话与工作台

### `dialog`

存储普通对话会话与其绑定对象。

### `history`

存储对话中的消息内容、角色和事件信息。

### `workspace_session`

存储工作台会话，包括：

- 标题
- 用户
- 会话上下文
- 更新时间

## 反馈与使用记录

### `message_like`

保存用户对消息的反馈记录。

### 其他统计与配置表

仓库中还包含与模型、知识库、MCP、技能和工具相关的配置表，用于支撑平台的能力管理与运行时装配。

## 说明

如果你在代码中看到带有 `mcp_agent` 的旧命名，它更多是兼容旧数据结构的保留命名，不代表当前系统运行时采用了多 Agent 套娃架构。
