索引设置

这些设置改变文档入库方式，保存后需要重建受影响文件的索引。

需要重建索引
修改 chunk、embedding、图片索引、向量库或 GraphRAG 模式后，需要重建索引。Top K、Rerank、补检策略不需要重建。

GraphRAG 索引参数

仅在知识库模式为 RAG + GraphRAG 时生效，修改后需要重建图谱索引。

图谱索引
从纯 RAG 升级到 RAG + GraphRAG 时，不需要重做文本向量索引，但需要对已有 chunk 补建实体关系图谱。

查询设置

即时生效

GraphRAG 查询参数

链路预览

展示当前配置下，一次查询会怎么走。

1. 向量召回 query embedding 后从向量库召回 Top K=5 个 chunk。
2. 图谱扩展 从 query 和命中 chunk 抽实体，到 Neo4j 扩展 1-hop / 2-hop 关系路径。
3. Rerank 与过滤 使用 gte-rerank-v2 重排，保留 Top 3，过滤低于 0.70 的结果。
4. 上下文回填 文本证据和关系路径一起交给 Agent 生成回答。

返回给 Agent 的证据

混合模式下不丢弃 RAG 结果，GraphRAG 作为关系补充。
