# Legacy Docs Map

## 目的

帮助 Agent 在需要历史上下文时，知道去哪里找旧资料，同时不把旧资料误当成当前主入口。

## 当前主入口

- `docs/architecture/README.md`
- `docs/architecture/plans/superpowers-legacy-migration-inventory.md`

## 历史资料入口

- `docs/superpowers/README.md`
- `docs/superpowers/specs/`
- `docs/superpowers/plans/`

## 使用规则

1. 默认先读 `docs/architecture/`
2. 只有在需要历史设计细节、旧计划、旧命名背景时，再读 `docs/superpowers/`
3. 如果旧资料与新方案冲突，以 `docs/architecture/` 为准
