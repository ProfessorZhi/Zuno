# Spec Coding

## 定义

这个仓库的 `spec coding` 指：

```text
先从正式文档里拿到目标、边界和验收标准，
再把这些内容转成开发过程中的真实目标，
直到验收点达成为止。
```

## 允许的工作模式

- 多 Agent 模式
- 目标模式
- 文档驱动开发
- 验收点驱动收口

## 推荐目标来源

- `docs/architecture/specs/`
- `docs/architecture/plans/`
- `docs/architecture/decisions/`
- `docs/README.md`
- `README.md`

## 推荐流程

1. 先读相关正式文档
2. 抽出目标、边界、验收点
3. 必要时开启多 Agent 模式分解任务
4. 用目标模式持续跟踪关键目标
5. 在代码、测试、文档三处同时完成收口
6. 验收点未达成前，不视为完成

## 工作流规则

- 多 Agent 模式是允许使用的标准开发手段
- 目标模式是处理验收驱动任务的标准手段
- 文档验收点可以直接作为目标
- 不允许把“已分析”误当成“已完成”

## 什么时候适合开多 Agent

- 任务跨多个子系统
- 既要看代码，又要看文档，又要做验证
- 需要并行梳理旧资产和新方案
- 一个主任务可以自然拆成互相独立的子任务

## 什么时候适合用目标模式

- 文档里已经写明验收标准
- 任务不能停在分析阶段
- 需要持续检查“是否真正完成”
- 需要把阶段目标串起来推进

## 结果要求

`spec coding` 不以“有想法”为完成，而以“满足验收点”为完成。
