一段话说明整个项目

Zuno 是一个本地优先的 Agent Workspace。Vue Web 和 Electron 负责产品交互，FastAPI 提供认证、 DTO、SSE 和 Application Service 边界；所有对话进入一个 GeneralAgent。GeneralAgent 在每次模型调用前由 ContextOrchestrator 按 token budget 组织上下文，在对话循环中按需选择 Knowledge、Action Tool、MCP 和 Skill， 通过 KnowledgeQueryService / GraphRAGQueryService 查询普通 RAG 与 GraphRAG Project，并在每轮之后把 raw events、 task summaries、structured memory candidates、evidence、citation 和 trace 写入可验证闭环。

Current / Foundation

  • Single GeneralAgent mainline
  • KnowledgeQueryService application boundary
  • GraphRAGQueryService / GraphRAGProjectSnapshot
  • minimal ContextOrchestrator
  • memory / capability foundation contracts

Target

  • mature Context / Memory Engine
  • ToolCard retrieval and Native BM25
  • multi-query + multi-retriever + RRF
  • full Evidence / Citation / Trace closure
  • explicit prepare_context -> agent_loop -> post_turn_commit graph

Future

  • Java business services
  • microservices
  • event-driven workers
  • product-level multi-agent mode
  • Coding Agent mode

Context / Memory Engine

主策略

Summary Compression + Structured Extraction

  • Sliding Window 只是 L1 recent window / token budget 保护。
  • Importance Filtering 只是 retention / selection 辅助策略。
  • Summary 不能替代 Raw Event Log。
  • Task Summary 和 Structured Memory 都必须带 source_event_ids。
  • External Knowledge 不是 Agent Memory。
  • Prompt Caching 是 compute-layer hint，不是 memory compression。

Context Layers

L0 Working Context L1 Recent Interaction Window L2 Task Summary Memory L3 Structured Long-term Memory L4 External Knowledge Raw Event Log

Native BM25 边界

Elasticsearch 是外部检索后端；BM25 是排序算法。Zuno 目标需要 NativeBM25Retriever，不能把 Elasticsearch 等同为 BM25 算法本体。

NativeBM25Index: tokenizer, inverted_index, term_freq, doc_freq, doc_len, avg_doc_len, idf NativeBM25Retriever: build(documents), search(query, top_k), explain_score(query, doc_id) score(q,d) = Σ idf(t) * tf(t,d) * (k1 + 1) / (tf(t,d) + k1 * (1 - b + b * |d| / avgdl)) k1 = 1.2 ~ 1.5 b = 0.75

GraphRAG query methods are basic, local, global, and drift; auto 是 router, not a fifth query mode.

Enhanced Retrieval Chain

User Query -> Query Understanding -> Query Variants -> Multi-retriever Recall Native BM25 Dense Vector Graph Local Community Global -> Deduplication -> RRF Fusion -> Optional Rerank -> Evidence Check -> Citation Builder -> Trace

Query variants 是 enhanced / expensive mode，不是默认每次开启。原始 query 必须保留。

RRF 默认 k = 60，是粗排融合，后续可接 optional rerank。输出 fusion_trace、rerank_trace 和 citation coverage。
