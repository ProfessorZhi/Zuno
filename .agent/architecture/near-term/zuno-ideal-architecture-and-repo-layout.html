Zuno Target Architecture: Local-first Single GeneralAgent Workspace

这份页面只解释近期目标架构，不是 Current proof。它把原来的多层概念图压成一条更容易讲清楚的主链路：任务进入 Zuno，系统选择能力，归一多来源证据，生成回答或产物，并把过程通过 Trace 事件展示给前端。

Local-first Single GeneralAgent ToolCard / Capability Registry Knowledge / GraphRAG Evidence / Citation / Trace / Eval

01 / Query Journey：从任务到交付物

参考 DeepSearchAgents 的清晰主链路，但使用 Zuno 的六层语言：产品入口层接收任务，API 创建 task / session，Single GeneralAgent 调用 Capability Selector，多来源结果进入 Evidence / Citation / Trace，最后返回回答、Markdown、PDF 或其他 Artifact。

1

产品入口层

Web / Desktop 只负责交互、上传、任务状态和事件展示，不拥有检索策略、图遍历或 provider 细节。

2

Typed API

FastAPI 接收 task、session、upload、artifact list / download 请求，并传播 request_id 与 trace_id。

3

Session Workspace

绑定 task id、session id、上传暂存、运行中间物、最终产物、事件索引和 trace。

4

Single GeneralAgent

唯一会话运行时执行 prepare_context -> agent_loop -> post_turn_commit，不默认拆成固定多 Agent。

5

Capability Selection

先检索 ToolCard，再按权限、健康、成本和任务相关性筛选，只注入选中的工具 schema。

6

RAG / GraphRAG 层

KnowledgeQueryService 统一 basic、local、global、drift；auto 是 router，不是第五种 query mode。

7

Evidence Trace

所有来源先归一成 evidence bundle、citation、coverage、latency、cost 和 fallback reason。

8

Answer / Artifact

返回流式回答，也可以生成 Markdown、PDF 或其他 Artifact，并通过 artifact_created 事件暴露给前端。

02 / 多来源 Evidence Channel

DeepSearchAgents 用 Tavily、MySQL、RAGFlow 和上传附件讲清了多来源任务。Zuno 不绑定这些具体工具，而把它们吸收成可替换的 capability / adapter。

公开网络

Tavily 这类 provider 映射为 Web Search Capability，输出 URL、source span 和 web evidence。

结构化数据

MySQL 这类连接器映射为 Structured Data Capability，输出 table / row evidence 和权限决策。

内部知识

RAGFlow 只能是 external RAG adapter；Zuno canonical 主线仍是 GraphRAGProject 与 GraphRAGQueryService。

上传附件

文件先属于 Session Workspace，再由 File Capability 抽取为 file evidence；它不是 Agent Memory。

生成产物

Markdown / PDF / report 由 Artifact Capability 生成，由 Platform 保存和发布。

边界：所有来源都可以进入本轮 Agent Context，但长期记忆必须经过 Raw Event Log、Summary Compression、Structured Extraction、source_event_ids 和权限检查。

03 / 三个支撑引擎

目标架构不靠“更多角色名”变成熟，而靠上下文、能力选择、知识检索和证据闭环变稳定。

Context / Memory Engine

调用前构造 ModelContextPacket；调用后写 Raw Event Log、Summary Compression、Structured Extraction 和 ContextTrace。

prepare_context -> ModelContextPacket post_turn_commit -> Raw Event Log -> Summary Compression -> Structured Extraction

Capability Registry

ToolCard 是能力检索语料；Native BM25 over ToolCards 是本地优先 baseline，optional vector search 只做增强。

Task / Context -> ToolCard Retrieval -> Native BM25 -> permission / health / cost -> selected schemas

Knowledge Router

Knowledge 层统一 RAG、GraphRAG、web evidence、file evidence 和 structured evidence；RRF 是默认 coarse fusion。

Query -> basic / local / global / drift -> BM25 / vector / graph / community -> dedup -> RRF -> Citation

04 / 会话隔离与长任务可观察

Zuno 可以学习 DeepSearchAgents 的 monitor 思路，但目标事件应是 typed runtime events，并进入 Trace / Observability，不成为新的业务层。

task_started workspace_created capability_selected tool_call_started evidence_ready artifact_created fallback_applied task_cancelled error
实现选择：SSE 或 WebSocket 都可以承载事件流。session_dir 和 ContextVar 是可选实现，不是架构身份；目标身份是 Session Workspace 与 Task Context。

05 / 必须保留的边界

Target not Current

本页是 Target / Proposed。Current 仍以 docs/architecture/current-architecture.md、代码、测试和 evidence 为准。

Single GeneralAgent

近期主线仍是 Single GeneralAgent Runtime。产品级多 Agent 模式仍是 Future，不是默认 runtime。

GraphRAG is Knowledge Capability

GraphRAG 向 GeneralAgent 返回 Knowledge Evidence，不接管对话，也不是第二套聊天 runtime。

Provider Neutral

Tavily、MySQL、RAGFlow 是学习样例。Zuno 目标写 Web Search、Structured Data、GraphRAG / RAG adapter。

Roadmap Owns Status

Phase 05、Phase 09 和 Execution Contract 只作为历史或路线图里的切片标签出现；active program 状态由 roadmap 和 .agent/programs/ 承担。

Domain Pack Retired

Domain Pack 只保留为 history / migration compatibility 语境；公开目标身份字段是 graphrag_project_id。
